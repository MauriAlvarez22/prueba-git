Git y Github

Probando Git y Github por primera vez

    Estructura
  • Inicializar el repositorios con git init. Recordar hacerlo dentro de la carpeta correcta.
  • "Iniciar sesion" utilizando el comnado git config user.name, este para el nombre y el git config user.email, para el coreo. Para hacerlo de froma global y no tenrer que hacerlo cada vez que inicializamos un repositorios se le agrega --global entre config y user, queadria asi git config --global user.(namo o email).
  • Vincular el repositorio local con el repositorio remoto, este se hace copiando la url que te aparce en git, ver imagen para entender. Para corroborar si todo salio bien, utilizamos este comando, git remote -v, y te apareceran dos url iguales.
  • Gurdar los archivos dentro del repositorio. Con el comando git status, se van a mostar todos los archivos que no tienen seguimiento. Para agregarlos e utiliza el comando git add, acompanado del nombre del archivo que queremos guardar.
  • Ahora tenemos que comitear los archivos. Esto se hace para poder subir los archuvos dentro de un paqute y de esta forma poder subir los archivos a Github. Para crear un commit de usa Esto git commit -m "un nombre que identifique este commit". Una vez comiteados le metemos un git push origin master y este commit se va al repositorio de Github.
